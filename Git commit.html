Git Commit

Git commit is a command used to save changes to the local repository. It captures a snapshot of the project's current state, allowing you to track changes over time.

Basic Usage

git commit -m "Your commit message here"

The -m flag allows you to include a commit message directly in the command. This message should be descriptive and explain the changes made.

Common Options

  • --amend: Modify the most recent commit. Useful for correcting mistakes or adding forgotten changes.
  • -a: Automatically stage files that have been modified and deleted, but not new files.
  • --no-edit: Use the existing commit message when amending a commit without opening an editor.

Example Workflow

git add .               # Stage all changes
